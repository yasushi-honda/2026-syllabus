システム開発B

土曜 9:20-12:30 後期 グローバルIT 2年次

担当教員

本田 康志（サポート講師）

授業概要

  • Java Servlet を使った Web アプリケーションを チーム開発 で制作する、実践科目です。
  • 開発環境は Eclipse を使用します。
  • WBS（作業分解構造）の作成、設計発表、開発、中間発表、最終発表会まで、本格的な開発プロセスを一巡します。
  • 評価は「個人の成果物」だけでなく、「チームへの貢献度」「メンバー相互評価」も含みます。

到達目標

知識・理解

  1. チーム開発における役割分担・WBS・進捗管理の考え方を理解している。
  2. Git/GitHub によるブランチ戦略・Pull Request・コードレビューの流れを理解している。
  3. Web アプリケーションの基本的なセキュリティ留意点を説明できる。

思考・判断

  1. 自分の担当範囲と、チーム全体の進捗との関係を判断できる。
  2. 詰まったときに「自分で調べる」と「チームに相談する」のバランスを判断できる。

スキル・表現

  1. チームでひとつの Web アプリ（Java Servlet）を作り上げられる。
  2. 自分の担当パートについて、設計意図と実装内容を発表できる。
  3. 他のメンバーのコードをレビューし、建設的なコメントができる。

関心・意欲・態度

  1. チームメンバーに敬意を持ち、協力する姿勢を持つ。
  2. 進捗・課題を正直にチームに共有できる。

使用テキスト・参考文献

特定のテキストは指定しません。授業内で用意する資料・サンプルコードを使用します。

参考: Apache Tomcat ドキュメント。

予習・復習

  • 予習: 自分の担当パートの実装に必要な情報を、事前に調べておくと授業時間を有効に使えます。
  • 復習: チームのリポジトリで他メンバーの変更を確認し、全体像を把握しましょう。

評価方法・基準

基本的にアプリケーションを作るにあたっての総合的なポイントを評価基準とします。

30%提出物 — 個人の成果物、コード品質
30%WBS等のチーム開発成果物 — WBS、設計書、進捗管理ドキュメント
40%メンバーの参加度 — 講師評価＋チームメンバーからの相互評価

その他(履修上の注意点)

  • 開発環境は Eclipse を使用します。
  • チームに迷惑をかけないこと が一番大切です。詰まったら早めに共有しましょう。「分からないまま黙っている」のがチーム開発で一番困ります。
  • 欠席する場合は、事前にチームメンバーと講師に必ず連絡してください。
