自然言語開発

新設科目 木曜 1・2限 IT・情報処理専攻(システム) 3年次

担当教員

本田 康志

非常勤講師 / ヤススシステム代表

「3年次のみなさん、ようこそ。この授業では、2026年現在の最前線——自然言語でソフトウェアを開発する技術を学びます。プロンプトエンジニアリングを超えた『コンテキストエンジニアリング』、AIエージェントを制御する『ハーネスエンジニアリング』、そして仕様を起点にした『Spec-Driven Development（SDD）』。これらは、AIと協働する開発者に求められる新しい専門スキルです。同時に大切にしたいのが 『責任あるコード』 という考え方。AIが高速にコードを生成できる時代だからこそ、何を作るか・なぜ作るか・どう検証するかを自分で判断できる開発者が求められています。まずは進級制作のレビューから始めましょう。みなさんの作品を一緒に振り返り、そこから新しい開発手法の旅を始めます。」

授業概要

  • GitHub Codespaces × Gemini CLI による完全クラウドベースの開発環境で、自然言語による本格的なソフトウェア開発手法を学ぶ、実践中心の専門科目です。ブラウザだけで受講でき、ローカルPCへのインストールは一切不要です。
  • 個人のGoogleアカウントだけで Gemini CLI の無料枠（60リクエスト/分・1,000リクエスト/日、Gemini 3 モデル、1Mトークンのコンテキスト）を使って学習します。追加課金は不要です。
  • 「Vibe Coding（直感的AI開発）」から始まり、Spec-Driven Development（SDD：仕様駆動開発）へと段階的にレベルアップ。プロトタイプ段階ではAIと自由に対話しながら素早く形にし、プロダクション段階では仕様書を起点に品質を担保する——この使い分けを体得します。
  • Gemini CLI の主要機能（GEMINI.md（プロジェクト別コンテキスト）、Plan Mode、MCP（Model Context Protocol）、Hooks（BeforeTool/AfterTool イベントによるエージェント制御）、Agent Skills、サンドボックス、カスタムコマンド）を実際に触りながら、AIエージェントの仕組みと制御方法を学びます。
  • コンテキストエンジニアリング（プロンプトを超えた全体的な情報設計）、ハーネスエンジニアリング（テスト・lint・レビューによるAI出力の制御基盤）など、2026年現在のAI開発の最前線を学びます。
  • 進級制作のレビューを起点に、自身のコードを客観的に評価し改善する力を養います。
  • 「責任あるコード」を貫くテーマとし、AIが生成するコードに対する検証・品質管理・倫理的判断の重要性を学びます。
  • PRD（製品要求仕様書）、ADR（アーキテクチャ決定記録）、TDD（テスト駆動開発）など、ソフトウェアエンジニアリングの基礎的なセオリーも実践の中で身につけます。

到達目標

知識・理解

  1. プロンプトエンジニアリング、コンテキストエンジニアリング、ハーネスエンジニアリングの違いと関係を説明できる。
  2. Spec-Driven Development（SDD）の概念と、TDD・PRD・ADRとの関係を理解している。
  3. AIコーディングエージェント（Gemini CLI 等）の仕組みと、ガイド（GEMINI.md 等の事前制御）・センサー（自動テスト等の事後検証）の概念を説明できる。
  4. 「責任あるコード」の原則（検証可能性、説明責任、倫理的配慮）を理解している。
  5. Git/GitHub Flowを用いたチーム開発プロセス（Issue, Branch, PR, Review, Merge）を説明できる。

思考・判断

  1. Vibe Codingが適切な場面とSDDが必要な場面を判断できる。
  2. AIが生成したコードの品質・安全性・保守性を批判的に評価できる。
  3. 仕様の曖昧さや矛盾を検出し、明確化するための質問を構築できる。
  4. テスト観点（正常系・異常系・境界値）を自ら設計できる。

スキル・表現

  1. AIエージェントを活用して、仕様に基づいたWebアプリケーションを開発できる。
  2. PRD・ADRを作成し、設計判断を文書化できる。
  3. テストを先に書き、実装を後から行うTDDの基本サイクルを実践できる。
  4. GitHub Flow（Issue→Branch→PR→Review→Merge）で開発を進められる。
  5. コードレビューにおいて、建設的なフィードバックを提供できる。
  6. 自身の設計判断とAI活用プロセスをプレゼンテーションで説明できる。

関心・意欲・態度

  1. AI開発の最前線の手法に興味を持ち、自ら調査・試行する姿勢を持つ。
  2. コードの品質と責任に対する意識を持ち、「動けばいい」で終わらせない姿勢を持つ。
  3. チーム開発において、他のメンバーのコードや意見を尊重し協力できる。
  4. 技術の急速な変化に対して、継続的に学び続ける意欲を持つ。

授業計画（全15回）

週 テーマ 主な内容・活動
1オリエンテーション＆進級制作レビュー授業の目的・進め方・評価方法。進級制作の相互レビュー（コードの可読性、設計、改善点を議論）。「責任あるコード」の導入——AIの時代に開発者に求められる資質。
2コンテキストエンジニアリング入門プロンプトエンジニアリングからコンテキストエンジニアリングへの進化。コンテキストウィンドウの仕組み、U字型精度カーブ、情報配置の戦略。実践：同じ課題をプロンプトのみ vs コンテキスト設計で比較。
3Gemini CLI × Codespaces 環境構築GitHub Codespaces 上で npx @google/gemini-cli を起動。個人Googleアカウント（OAuth）で認証。無料枠の仕組みを理解（60 req/min、1,000 req/日、Gemini 3、1Mコンテキスト）。GEMINI.md 初体験。動作原理（ReActループ、ツール呼び出し）。Vibe Coding体験：自然言語だけでWebアプリを作る。
4Vibe Codingの実践と限界Vibe Codingで小規模アプリを高速開発。生成されたコードの品質分析（動くけど保守できない問題、セキュリティリスク等）。「なぜVibe Codingだけでは不十分か」の体験的理解。
5PRD（製品要求仕様書）とADRPRDの書き方：ユーザーストーリー、機能要件、非機能要件。ADR（アーキテクチャ決定記録）：なぜその技術選択をしたかを記録する文化。AIにPRD/ADRのドラフトを作成させ、人間がレビュー・修正する演習。
6Spec-Driven Development入門SDDの概念：仕様が実行可能な成果物になる。SDDライフサイクル（定義→計画→分解→実行→デバッグ）。Gemini CLI の Plan Mode を使って「実行前に計画を立てる」サイクルを体験。Vibe Coding → SDD への移行。仕様ファイルの書き方。
7テスト駆動開発（TDD）基礎TDDの基本サイクル：Red → Green → Refactor。テスト観点の設計（正常系・異常系・境界値）。AIにテストを書かせ、人間がテスト観点を検証する演習。テストファーストの思考法。
8ハーネスエンジニアリング入門ハーネスの概念：AIの出力を制御する基盤。ガイド（GEMINI.md、.geminiignore、サンドボックス、ブランチ保護等の事前制御）とセンサー（GitHub Actions によるテスト・lint・ビルド、PRレビュー等の事後検証）。Gemini CLI の Hooks 機能（BeforeTool/AfterTool）で機密情報コミット阻止・ツール引数検証・監査ログを実装する実践。
9中間レビュー＆コードレビュー実践ここまでの成果を相互レビュー。GitHub PR を使ったコードレビューの実践。建設的なフィードバックの書き方。AIレビューと人間レビューの違いと使い分け。
10SDD実践プロジェクト①：設計チームまたは個人でSDD実践プロジェクト開始。仕様書の作成（AIと協働）。テスト計画の策定。アーキテクチャの決定とADR作成。Issue分解とGitHub Project活用。
11SDD実践プロジェクト②：実装仕様に基づくAIエージェント活用開発。Gemini CLI の MCP サーバー 連携で外部ツール（DB・API等）と接続する体験。TDDサイクルの実践。ハーネス（テスト・lint）による品質チェック。GitHub Flow（Branch→PR→Review→Merge）の実践。
12SDD実践プロジェクト③：検証と改善テスト結果に基づく品質評価。仕様と実装の差異分析。リファクタリングと技術的負債の管理。セキュリティ観点でのコードチェック。
13責任あるAI開発Anthropic Claude Constitution（CC0公開文書）を教材にした倫理ディスカッション。EU AI Act等の規制動向。AI生成コードの著作権・ライセンス問題。「開発者の責任」とは何かを考える。
14プロジェクト最終仕上げ最終調整・テスト・ドキュメント整備。発表準備。プロジェクト全体のADR・振り返りの記録。
15成果発表＆総括プロジェクト成果発表（技術選択の根拠、AI活用プロセス、品質管理の工夫を含む）。相互評価。前期の振り返り：「プロンプト→コンテキスト→ハーネス」の進化を実感。後期への接続。

評価方法・基準

AIエージェントを効果的に活用しつつ、コードの品質と設計判断に責任を持てるかを重視します。

25%仕様・設計の品質 — PRD/ADR/仕様書の明確さ、テスト設計の網羅性、設計判断の妥当性。
25%実装・コードの品質 — コードの動作、可読性、保守性、セキュリティ配慮。テストの充実度。
20%AI活用の高度さ — コンテキスト設計の工夫、ハーネスの構築、AIへの効果的な指示、出力の検証プロセス。
15%開発プロセス — GitHub Flowの実践（Issue/PR/Review）、継続的な進捗管理、チームでの協働。
15%「責任あるコード」の実践 — AI生成コードの検証姿勢、倫理的配慮、設計判断の説明責任。

その他（履修上の注意点）

  • GitHubアカウント と 個人のGoogleアカウント（Gemini CLI OAuth認証用）が必要です。Googleアカウントは無料枠で利用するため追加費用は発生しません。
  • 開発環境は GitHub Codespaces を使用するため、学生のデバイスはブラウザが使えれば何でも構いません（Chromebook・iPad・古いノートPC可）。
  • 進級制作のコード・成果物を第1回で使用します。アクセスできる状態にしておいてください。
  • 体調やメンタル面で不安がある場合は、無理せず担当教員または学校の相談窓口に相談してください。
  • この授業で学ぶスキルは、現在のIT業界で急速に需要が高まっている分野です。授業外でも積極的に実践し、ポートフォリオとして活用してください。
