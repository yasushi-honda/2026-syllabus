はじめてのAI体験

Week 2 木曜 3・4限
テーマ
Geminiに自分で質問を投げて、応答をじっくり観察する
キーワード
個人演習 / 質問のバリエーション / 応答のゆらぎ / 観察ノート
必要な準備
Googleアカウントでログインできる状態・授業用のメモ（紙でも Google Docs でも可）

この回のゴール

  • 前回一緒にやったGeminiに最初の質問を投げる体験を、自分ひとりでもう一度できる
  • いろいろなタイプの質問（事実／意見／手順／相談）を自分で考えて投げてみる
  • 同じ質問を何度か投げると答えがちょっと違うことに気づく
  • 聞き方を変えると応答の雰囲気も変わることを観察する
  • 「自分が面白い／意外／便利」と感じた応答を言葉でメモできる

授業の流れ

2コマ連続（90分 + 90分 = 計180分）。途中休憩あり。今日は自分のペースで進めることを大事にします。

1限 前半 約40分

前回の振り返り＆今日のゴール

前回のおさらい（10分）

  • 前回の「一番印象に残ったGeminiとの会話」を2〜3人に口頭で共有してもらう
  • 「AIは毎回ちょっと違うことを言う」感触を思い出す
  • 家でGeminiを触ってみた人の体験談もあれば歓迎（触ってない人でも大丈夫）

今日の進め方（10分）

  • 今日は個人演習中心——自分のペースで質問を投げる時間
  • 上手くいかなくても大丈夫。「変な応答が返ってきた」も立派な観察結果
  • タイピングが不安な人は、短い質問から始めてOK。スマホの音声入力を使ってもOK

「観察する」ってどういうこと？（20分）

  • 今日のキーワードは観察。答えを受け取って「はい次」ではなく、応答をじっくり読むこと
  • 見るポイントの例:
    • 答えの長さ（1行？10行？）
    • 使っている言葉づかい（ていねい？くだけてる？）
    • 自信ありげか、「たぶん」「かもしれない」が多いか
    • 答えのなかに具体例が入っているか
  • 観察のためのメモ用紙（もしくはGoogle Docsの新規ファイル）を用意しておく
プログラミングとのつながり: この「観察する→気づく→メモする」という動きは、これからプログラムを書くときにも毎回やることです。今日はその筋トレだと思ってください。
1限 後半 約50分

4種類の質問を投げてみる

Geminiを開く（5分）

  • gemini.google.com にアクセス
  • Googleアカウントでログインが切れていたら再ログイン
  • 画面下部の入力欄が表示されていればOK（UIは変更される場合あり——迷ったら講師に声をかけてください）

パターン①: 事実を聞く（10分）

  • 例: 「富士山の高さは？」「ラーメンの発祥はどこ？」「大阪から東京までの新幹線の所要時間は？」
  • 自分で3つ考えて投げる。答えをそのままコピーしてメモに貼る

パターン②: 意見を聞く（10分）

  • 例: 「夏と冬、どっちがおすすめ？」「猫と犬、どっちが飼いやすい？」
  • AIは「どちらにも良さがあります」のようにどっちつかずの答えをしがち。その癖を観察する

パターン③: 手順を聞く（10分）

  • 例: 「カップ麺の上手な作り方を5ステップで」「自転車のパンク修理の流れを教えて」
  • 番号付きで整理された応答になりやすい——その形式のちがいを観察する

パターン④: 相談する（15分）

  • 例: 「朝どうしても起きられない、何をしたらいい？」「はじめて自炊するときの最初の一品は？」
  • AIがこちらの状況（生活リズム、予算、好み）を聞き返してくるか、決めつけて答えてくるかを観察
  • 個人情報（本名・住所・電話番号・学校の正式名称など）は入れない——入力した内容はAI改善のために使われる可能性があるため、個人を特定できる情報は避ける
休憩 10分
2限 前半 約50分

「ゆらぎ」と「聞き方」の実験

実験A: 同じ質問を3回投げる（20分）

  • パターン①〜④の中から、自分が一番気に入った質問を1つだけ選ぶ
  • その質問を、新しいチャットで3回投げる（前の会話を閉じて毎回やり直すのがコツ——同じチャットの中で続けると、前の会話に引きずられる）
  • 3つの答えを並べてメモ。どこが同じで、どこが違うかを観察
  • AIには「ゆらぎ」がある——同じ質問でも毎回少しずつ違う答えを返す。これを体感する

実験B: 聞き方を変える（25分）

  • 同じ「聞きたいこと」を、3通りの聞き方で試す:
    • ざっくり版: 「〜〜について教えて」
    • ていねい版: 「〜〜について、小学生にもわかるように、例を2つ入れて5行くらいで教えてください」
    • キャラ指定版: 「あなたは〜〜の先生です。〜〜について教えて」
  • 答えの長さ・語り口・具体例の数がどう変わるかをメモ

ミニ気づきメモ（5分）

  • 実験A/Bで一番驚いたことを、自分の言葉で1〜2行書く
2限 後半 約30分

共有タイム＆「不思議な応答」の扱い方

近くの人と見せ合う（15分）

  • 2〜3人で、今日の実験A/Bで面白かった応答をひとつずつ見せ合う
  • 「自分なら違う聞き方にするかも」と思ったら、アイデアを交換する

変だった応答・間違っていた応答（10分）

  • 今日のやり取りで「ん？ちょっと違うかも」と感じた応答があれば共有
  • AIの答えは堂々としているのに間違っていることがある——これを今後は「ハルシネーション（もっともらしい作り話）」と呼ぶ。第9回で詳しく扱う予定
  • 今日は「気づけた」ことを褒める。疑う力は大事なスキル
合言葉: 「AIはすごい、でも完璧じゃない」——この感覚を今日のうちに体で覚えてしまおう。
まとめ 約10分

  • 今日のメモは次回以降も使います。捨てずに残しておくこと

課題・次回までにやること

必須
観察メモの保管

今日書いた「4種類の質問の応答」「実験A/Bの結果」「気づき」を捨てずに保管する。紙ならスマホで写真を撮っておくと安心。

推奨
家でも質問を5つ投げてみる

今日は学校で投げた質問と違う話題で、自分が気になるテーマを5つ投げてみる。個人情報は入れないこと。

推奨
「ん？」と思った応答をスクショ

明らかに変な答え／事実と違いそうな答えを見つけたら、スクリーンショットを残しておく。次回以降、「AIの限界」の教材として使えます。
