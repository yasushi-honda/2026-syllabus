HTMLの構造を学ぶ

Week 2 火曜 1・2限
テーマ
HTMLの骨組みを理解し、サンプルを読んで自分のページを拡張する
キーワード
head / body / 見出し / 段落 / リスト / リンク / 画像 / 読んで理解する
必要な準備
第1回で作成した index.html（自分のリポジトリに残っていること）・Codespacesが起動できる状態

この回のゴール

  • HTMLの基本構造が <head> と <body> の2つに分かれていることを理解する
  • 見出し・段落・リスト・リンク・画像、5つの基本タグの役割を自分の言葉で説明できる
  • 配布されたサンプルコードを読んで理解する（コピペして動かす、ではない）
  • 第1回の自己紹介ページを、今日学んだタグでプロフィールページに拡張する
  • 「なぜこのタグを使ったのか」を自分のコメントで残せるようになる

授業の流れ

2コマ連続（90分 + 90分 = 計180分）。途中休憩あり。

1限 前半 約40分

前回の振り返りとHTMLの全体像

前回の振り返り（10分）

  • 第1回で書いた index.html をCodespacesで開き直す
  • ターミナルで python3 -m http.server 8000 → 「ポート」タブから 「エディターでプレビュー」 で Simple Browser 表示（第1回と同じ手順）
  • 詰まったら Copilot Chat に「index.html を Simple Browser で開いて」 と頼んでOK——ここで時間を使いすぎない
  • GitHub Studentプランの申請状況をクラス全体で共有（承認メール到着者は手を挙げる）

HTMLの骨組みを図で理解する（20分）

  • HTMLは大きく <head> と <body> の2つに分かれる
  • <head>：ブラウザへの指示書——タイトル、文字コード、スタイル指定など「画面には出ないけれど大事な情報」
  • <body>：画面に表示される中身——見出し、文章、リスト、画像、リンクなど
  • 第1回で書いた <!DOCTYPE html>・<html>・<head>・<body> が「入れ物」で、中身はこれから増やしていく

今日覚える5つのタグ（10分）

  • <h1>〜<h6>（見出し）——ページの章立てを作る
  • <p>（段落）——文章のかたまり
  • <ul> / <li>（箇条書きリスト）——「好きなもの」「できること」を並べる
  • <a href="...">（リンク）——他のページに飛ぶ
  • <img src="..." alt="...">（画像）——プロフィール写真やアイコンを表示する
1限 後半 約50分

サンプルコードを「読んで理解する」

授業用サンプルHTMLを開く

  • 配布するサンプル（samples/profile-sample.html）をCodespacesで開く
  • まず「写経」ではなく「読む」——上から順に、どのタグが何を表示しているかを目で追う
  • 「このタグで、ブラウザにこう表示されているはず」を頭の中で予想してから、実際の表示と照らし合わせる

タグと表示の対応を1行ずつ確認する

  • <h1>山田 太郎</h1> → ブラウザに大きな見出しが出る
  • <p>情報処理科の2年生です。</p> → 段落として表示される
  • <ul><li>...</li></ul> → 箇条書きに変わる
  • <a href="https://...">GitHub</a> → 青い下線付きのリンクになる
  • <img src="..." alt="..."> → 画像が表示される（読み込めないと alt のテキストが出る）

ペアで説明し合う（最後の10分）

  • 隣の人と、「この行は何をしているか」をお互いに言葉で説明する
  • 言葉に詰まったら、それがまだ理解しきれていない場所
  • お互いに聞き合うことで「分かっているつもり」が見つかる
読む力が書く力を育てる: プログラミングは「書く」より先に「読む」が上手になります。サンプルコードを「もしこの行を消したらどうなる？」「もしこの属性を変えたらどうなる？」と問いながら読むのがコツです。
休憩 10分
2限 前半 約50分

自分の index.html をプロフィールページに拡張する

今日のミッション

  • 第1回で書いた自己紹介ページを、今日学んだ5つのタグで拡張する
  • サンプルを「コピペ」せず、サンプルの構造を真似て自分の内容で書く
  • 打ち間違いも含めて自分の手で打つ——先週と同じ方針

拡張する項目（最低限）

  1. <h1> で名前、<h2> で「自己紹介」「好きなもの」「リンク集」のような小見出しを3つ作る
  2. 自己紹介セクションに <p> で2〜3行の文章を書く
  3. 「好きなもの」セクションを <ul> と <li> で箇条書きにする（3つ以上）
  4. 「リンク集」に <a href="..."> で自分のGitHubプロフィールページへのリンクを1つ作る
  5. プロフィール画像を <img src="..." alt="..."> で1つ入れる（アバター画像URLでOK）

進度差のある人へ

  • 早く終わった人は自由に拡張——見出しを増やす、リストを入れ子にする、画像をもう1枚、など
  • 手が止まった人はサンプルをもう一度読むことから戻る。焦らなくて良い
  • クラス全体で同じ完成形を目指さない——各自のプロフィールが一番大事
2限 後半 約30分

コメントで「構造」を自分の言葉で残す

各セクションに「なぜこのタグか」を書く

  • 第1回と同じ <!-- コメント --> 形式で、セクションごとに意図を残す
  • 例: <!-- ここから好きなものリスト。ul は順序が関係ない箇条書きなので ol ではなく ul にした -->
  • 「同じ結果になる別のタグ」も意識する——なぜ <h1> を繰り返さず <h2> を使ったのか

ブラウザで最終確認

  • Simple Browser で自分のプロフィールが表示されることを確認（サーバー起動手順は第1回と同じ、ファイル保存後は更新ボタンで再読み込み）
  • リンクをクリックしてGitHubに飛ぶか確認
  • 画像が表示されない場合は alt のテキストが出ていればOK（URLの綴り確認は次回のデバッグ回で扱う）
AIとの付き合い方（今週）: Copilotが補完を出してきたら、「自分が書こうとしていたもの」と比べる時間にしてください。内容が違うなら自分の意図を優先、同じなら一度はEscで消してから自分で打ってみる——これが「使う・使わない」を判断する練習です。
