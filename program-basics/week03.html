CSS入門：色とレイアウト

CSSって何？（20分）

  • HTMLが「意味の構造」を表すなら、CSSは「見た目の指示書」
  • 同じHTMLでも、CSSを変えればまったく違うデザインになる——「内容と見た目の分離」が現代Webの基本
  • CSSの基本パターンは セレクタ { プロパティ: 値; } の3点セット
    • セレクタ: どのHTML要素に対してか（例: h1、p、body）
    • プロパティ: 何を変えるか（例: color、font-size、margin）
    • 値: どう変えるか（例: red、20px、1em）
  • 例: h1 { color: navy; } は「すべての <h1> の文字色を navy にする」という指示

CSSの「書く場所」3パターン（10分）

  • ① インライン: HTMLタグの style 属性に直書き（例: <h1 style="color:red">）——最後の手段。乱用しない
  • ② <style> タグ内: HTMLの <head> 内に書く——お試しはこれが速い
  • ③ 別ファイル（外部CSS）: style.css を作って <link rel="stylesheet" href="style.css"> で読み込む——本授業では基本このやり方
1限 後半 約50分

style.css をつくって読み込む

新しいファイルを作る

  • Codespaces のエクスプローラーで index.html と同じ階層に style.css を新規作成
  • index.html の <head> 内に <link rel="stylesheet" href="style.css"> を追加
  • これで index.html から style.css が読み込まれるようになる

最初の1行を手で書く

  • style.css に次の1行を打つ: 
    h1 { color: navy; }
  • 保存して Simple Browser を更新——<h1> の文字色が変わるはず
  • 変わらなかったら: ① <link> の href のスペル ② ファイルが保存されているか ③ Simple Browser を再読み込みしたか、を順に確認

セレクタを増やす（タグセレクタ中心）

  • body { background-color: #fafafa; }——ページ全体の背景色を変える
  • p { color: #333; line-height: 1.7; }——段落の文字色と行間を調整
  • a { color: teal; }——リンクの色を変える
  • 1行ずつ書いて保存→ブラウザで確認を繰り返す。まとめて書かない
色の書き方は3種類: ① 名前（red、navy、teal など）／② # + 16進数（#fafafa、#333）／③ rgb(255, 0, 0)。今日は 名前と # 16進数の2つを覚えればOK。
休憩 10分

フォント・余白で「読みやすさ」をつくる

フォントを指定する

  • body { font-family: "Helvetica Neue", Arial, sans-serif; }——半角英数のフォント指定
  • 日本語を含む場合: body { font-family: "Hiragino Sans", "Yu Gothic", sans-serif; } のように複数候補をカンマで並べる
  • fallback（フォールバック）の考え方: 最初の候補が無ければ次、それも無ければ最後の sans-serif（ゴシック系）／serif（明朝系）に落ちる
  • font-size も試す: h1 { font-size: 36px; } や p { font-size: 16px; }

余白で「呼吸」を入れる

  • CSSのボックスモデル: すべてのHTML要素は「内容 → padding → border → margin」の同心円構造で囲まれている
  • margin: 要素の外側の余白（他の要素との距離）
  • padding: 要素の内側の余白（枠線と内容との距離）
  • 試す: body { margin: 0; padding: 24px; } / h1 { margin-bottom: 16px; } / ul { padding-left: 24px; }

1色だけルール

  • 今日のミッションは「自分らしい1色」を決めて、それを見出しとリンクの色に統一すること
  • カラフルにしすぎない——多色は混乱を生む。制約があると個性が出る
  • 色名で困ったら参考リンクの「MDN — color」で名前付きカラー一覧を見る
変化は最小単位で: CSSは「1行書いて、保存して、ブラウザで確認」が基本リズム。複数の変更をまとめてからブラウザで見ると、どれが効いてどれが効いてないかが分からなくなります。
2限 後半 約30分

Copilotの提案を「採用するか・しないか」自分で判断する

あえてCopilotに聞いてみる時間

  • これまでは「まず手で書く」だったが、今日は最後にCopilotの提案を意識的に観察する時間をとる
  • style.css でコメントを書いて待つ: 
    /* h2 を青系の見出しっぽく見せたい */
  • Copilotが補完を出してきたら、Tabで採用する前に 提案の中身を読む

判断の3ステップ

  1. 読む: 提案されたプロパティと値を1つずつ見て、「これは何を変えるルールか」を自分の言葉で説明できるか確認
  2. 比べる: 自分が書こうとしていた内容と、Copilotの提案はどう違うか
  3. 決める: 採用なら Tab、不採用なら Esc。「迷ったら不採用」でOK——意味の分からないコードは入れない

採用/不採用の理由をコメントで残す

  • 採用したら: /* Copilot提案を採用: 青系で揃えたかったので */
  • 不採用にしたら: /* Copilot提案を却下: 値の意味が分からなかったので */
  • これが評価でいう 「AIの批判的活用」の最初の一歩
「動けば正解」ではない: Copilotの提案で見た目が良くなっても、意味の分からないコードを残すのは負債。後で修正できなくなります。今日は「分からないものは入れない」の練習日。
まとめ 約10分
