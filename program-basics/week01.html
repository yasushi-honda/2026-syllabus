オリエンテーション＆はじめてのHTML

Week 1 火曜 1・2限
テーマ
ガイダンス＋はじめて手で書くHTML
キーワード
GitHub Studentプラン / Codespaces / HTML基本タグ / 手で書く
必要な準備
GitHubアカウント（ログイン可能な状態）・学校のメールアドレス（Studentプラン申請用）

この回のゴール

  • 授業の全体像（目的・進め方・評価方法）を理解する
  • 「まず手で書く」「AIは補助」という本授業の基本姿勢を理解する
  • GitHub Studentプランに申請し、GitHub Copilotが使える準備をする
  • GitHub Codespacesを起動し、ブラウザ上で開発環境を開けるようにする
  • 自分の手でHTMLの基本タグを1つずつ打ち込み、自己紹介ページを表示させる
  • 「タグを書く → ブラウザで確認する」の最小サイクルを体験する

授業の流れ

2コマ連続（90分 + 90分 = 計180分）。途中休憩あり。

1限 前半 約40分

オリエンテーション

教員自己紹介・授業の位置づけ

  • 本田の自己紹介（実務でのAI活用、なぜこの授業を作ったか）
  • 「プログラム基礎 I」がITビジネスコースでどんな役割を持つか
  • 2〜4年次の混在クラスであること——各自のペースを尊重する方針

シラバスの説明

  • 全15回の流れを俯瞰（HTML → CSS → JavaScript → ミニ完成 × 3）
  • 評価方法の確認（理解と説明力30%、完成度25%、デバッグ力20%、AIの批判的活用15%、主体性10%）
  • 「動いた」だけでなく「なぜ動くかを自分の言葉で説明できる」ことを最重視する

この授業の基本姿勢

  • 「まず手で書く」——AIが何でも書いてくれる時代だからこそ、自分で一行ずつ書く時間を大切にする
  • AI（GitHub Copilot等）は補助として使う。使っていい時間・あえて使わない時間を意識的に切り替える
  • 「バグ修正ワーク」で、AIにわざとバグ入りコードを生成させて直す練習もする
1限 後半 約50分

GitHub Studentプラン＆Codespacesを起動する

GitHubアカウントの確認

  • アカウントを持っていない学生はその場で作成（教員がサポート）
  • ユーザー名は今後ずっと使うもの。本名に近い形が推奨（就職活動でも使える）

GitHub Studentプランの申請

  • GitHub Student Developer Pack にアクセス
  • 学校のメールアドレス・学生証を使って申請
  • 審査通過後、GitHub Copilot Student（学生向け無料プラン）や、Codespacesの月180コア時間枠が無料で使えるようになる
  • ※審査に時間がかかる場合あり。通過するまでは無料枠で進める

GitHub Codespacesの起動

  • 授業用リポジトリを各自フォーク
  • 「Code」ボタン →「Codespaces」→「Create codespace on main」
  • ブラウザ上にVS Codeの画面が立ち上がることを確認
  • 左側のファイルツリー、真ん中のエディタ、下のターミナルの位置を確認
Codespacesを選ぶ理由: ローカルPCに何もインストールせず、ブラウザだけで開発できるため、家でも学校でも同じ環境で作業できます。学校のPCが変わっても、自宅のPCが変わっても、GitHubにログインすれば続きから再開できます。
休憩 10分
2限 前半 約50分

はじめてのHTML——自己紹介ページを手で書く

index.html を空のファイルから作る

  • Codespacesで新規ファイル index.html を作成
  • コピペせず、1文字ずつ自分で打ち込む——これが今日の一番のポイント
  • 打ち間違いも含めて全部が学び。エラーが出たら「なぜ？」を考える

段階的に書くHTMLタグ

  1. <!DOCTYPE html> でHTMLであることを宣言
  2. <html>、<head>、<body> の骨組み
  3. <title> でブラウザのタブに表示される文字を書く
  4. <h1> で自分の名前を大見出しとして書く
  5. <p> で「はじめまして」などの自己紹介文を書く

ブラウザで表示してみる（3ステップ）

  1. Codespacesのターミナルで python3 -m http.server 8000 を実行（簡易サーバーを起動）
  2. ポートが自動転送された通知、または画面下の 「ポート」タブ から 8000 番の行を見つけて 「エディターでプレビュー」（画面アイコン）を選び Simple Browser を開く。「ブラウザーで開く」（地球儀アイコン）は外部ブラウザなので今日は使わない
  3. プレビュー画面に自分の書いたHTMLが表示されたら成功。ファイルを保存したら Simple Browser の更新ボタン で再読み込み
  • 「自分が書いた文字が、ブラウザに表示されている！」——この感動を味わう
  • 文字を変えて保存 → 更新ボタンで再読み込み、のサイクルを体験
詰まったら Copilot Chat に頼んでOK: コマンドが動かない・ポートが見つからない・Simple Browser の開き方が分からない——そんな時は Copilot Chat に 「index.html を Simple Browser で開いて」 と日本語で頼んでください。必要なコマンド実行と起動を代わりにやってくれることが多いです（Copilotの応答が期待通りでない場合は講師に声かけを）。今日の最優先は「自分が書いた HTML が画面に出る瞬間を体験すること」。手順の丸暗記は次回以降で構いません。
2限 後半 約30分

タグを少し増やしてみる＆なぜこう動くかを考える

もう少しタグを追加する

  • <ul> と <li> で「好きなもの」をリストにする
  • <a href="..."> で好きなサイトへのリンクを1つ追加
  • それぞれ「どのタグが何の役割をしているか」を自分の言葉で言えるようにする

自分の言葉でコメントを書く

  • HTMLのコメントは <!-- コメント -->
  • 自分が書いた各ブロックに「この部分は◯◯を表示している」とコメントを残す
  • このコメントが、次回以降の自分への手紙になる
AIとの付き合い方の最初の一歩: Copilotが使える状態になっていても、今日はできるだけ自分で書くことを意識してください。Copilotの補完が表示される場合は、Escキーで消して自分で続きを打つ練習をしましょう（Studentプラン審査中でCopilotがまだ有効でない人は、そのまま手で書く時間として活用してください）。
まとめ 約10分
