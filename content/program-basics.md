## 7. 担当教員からのメッセージ
プログラミングは、焦らず、ステップバイステップで学ぶのが一番の近道です。この授業では、まず**自分の手で一行ずつコードを書いて動かす**ことを大切にします。AIが何でも書いてくれる時代だからこそ、「なぜ動くのか」を自分の言葉で説明できる力が、これから本当に求められるスキルです。

とはいえ、AIを完全に避けるわけではありません。みなさんはGitHub Studentプランで**GitHub Copilot**を使える状態にあります。授業では「手で書く時間」と「AIに頼ってよい時間」を意識的に使い分けながら、AIの提案を**批判的に評価できる目**を養います。失敗も間違いも、すべて学びです。一歩ずつ、着実にいきましょう。

## 8. 授業概要
- HTML/CSS/JavaScriptを**自分の手で書きながら**、Webプログラミングの基礎を段階的に学ぶ入門科目です。
- 開発環境は **GitHub Codespaces** を使用します。ブラウザだけで完結し、ローカルPCへのインストールは不要です。学生は全員 **GitHub Studentプラン** を取得し、CodespacesとGitHub Pagesを無料で活用します。
- **「まず手で書く」**を基本姿勢とし、AI（GitHub Copilot等）は補助的に使います。AIがコードを自動生成できる時代だからこそ、**コードの意味を自分で理解する力**を最優先に養います。
- 2-3週ごとに小さな完成体験（GitHub Pagesへの公開）を設け、達成感とモチベーションを維持しながら学習を進めます。
- **「バグ修正ワーク」**を複数回設けます。AIにわざとバグ入りのコードを生成させ、それを自分で読み解いて修正する演習を通じて、コードを読む力・AIを批判的に評価する力を養います。
- 2〜4年次の混在クラスであることを踏まえ、各自のペースを尊重します。経験者にはより発展的な課題を、初学者には丁寧なサポートを行います。

## 9. 到達目標

### 知識・理解
1. HTML/CSS/JavaScriptの基本構文と、それぞれの役割を説明できる。
2. 変数、条件分岐、繰り返し、関数、配列、DOM操作の基本概念を理解している。
3. Gitによるバージョン管理の基本概念（add/commit/push）と、GitHub Pagesによる公開の流れを説明できる。
4. レスポンシブデザインの考え方（PCとスマホでの表示の違い）を理解している。
5. AIが生成するコードには誤りが含まれうることを理解し、その理由を説明できる。

### 思考・判断
1. コードを一行ずつ読み解き、何をしているか自分の言葉で説明できる。
2. エラーメッセージを手がかりに、問題の原因を推測し対処を試みることができる。
3. AIの提案に対して「採用するか・しないか」を理由とともに判断できる。
4. 「いま手で書くべきか、AIに聞くべきか」を場面に応じて判断できる。

### スキル・表現
1. GitHub Codespacesを使って、HTML/CSS/JavaScriptで意図したWebページを作成できる。
2. Git/GitHubの基本操作を行い、自分の作品をGitHub Pagesで公開できる。
3. AIが生成したコードのバグを発見し、修正できる。
4. 自分が書いたコードの動作を、コメントや言葉で説明できる。

### 関心・意欲・態度
1. 「自分の手で動くものを作る」喜びを大切にし、主体的に取り組む。
2. エラーや失敗を学びの機会と捉え、粘り強く試行錯誤する。
3. AIに頼り切らず、自分で考え検証する姿勢を持つ。

## 10. 授業計画（全15回）

| 週 | テーマ | 主な内容・活動 |
|---|---|---|
| 1 | オリエンテーション＆はじめてのHTML | 授業の目的・進め方・評価方法。GitHub Studentプランの確認、Codespacesの起動方法。**自己紹介ページを手で書く**体験（HTMLの基本タグを1つずつ入力）。 |
| 2 | HTMLの構造を学ぶ | HTMLの基本構造（head/body）、見出し・段落・リスト・リンク・画像。サンプルコードを**読んで理解する**練習。プロフィールページを手で拡張。 |
| 3 | CSS入門：色とレイアウト | CSSセレクタの基本、色・フォント・余白を手で書く。Copilotが提案してきたコードを「採用するか・しないか」自分で判断する練習。 |
| 4 | 🎯ミニ完成① プロフィールサイト公開 | レイアウト調整。GitHub Pagesでの公開方法（リポジトリ作成→公開設定）。**自分の作品が世界に公開される体験**。公開URLを取得。 |
| 5 | レスポンシブ＆📱スマホ実機確認＆🔧バグ修正ワーク① | 公開したURLを**QRコードに変換してスマホで読み取り**、実機で表示崩れを確認。「PCでは動いてもスマホで違う」を体験。メディアクエリの基本を手で書く。AIに「バグ入りスマホ対応CSS」を生成させ、修正する演習。 |
| 6 | JavaScript入門：変数と出力 | JavaScriptの基本（変数、データ型、文字列・数値）。`console.log`での動作確認。簡単なおみくじページを手で書く。 |
| 7 | 条件分岐＆🔧バグ修正ワーク② | if文の基本を手で書く。簡単な診断ゲームの制作。AIに「バグ入りif文コード」を生成させ、エラーを発見・修正する演習。 |
| 8 | 🎯ミニ完成② 小ゲーム公開 | ロジックを統合してGitHub Pagesに公開。スマホでも動作確認。**2作目の公開体験**。 |
| 9 | 繰り返しと配列 | for文、配列の基本を手で書く。リスト表示アプリの制作。複数のデータを扱う考え方を理解する。 |
| 10 | 関数とコード整理＆🔧バグ修正ワーク③ | 関数の定義と呼び出しを手で書く。コードを整理する意義。AIに「バグ入り関数」を生成させ、読み解いて修正する演習。 |
| 11 | DOM操作とイベント | HTMLの要素をJavaScriptで操作する基礎。クリックイベント。簡単なクリックゲームを手で書く。 |
| 12 | 🎯ミニ完成③ インタラクティブ作品公開 | DOM操作を統合して動きのある作品を制作。GitHub Pagesに公開。**3作目の公開体験**。 |
| 13 | デバッグ集中回（AIなしで直す） | エラーメッセージを読む練習。あえて**AI（Copilot）を使わずに**自力でバグを修正する体験。エラーへの向き合い方を学ぶ。 |
| 14 | 自由制作 | これまで学んだ技術を使って自由に作品を制作。AIの使い方は**各自が自分のルールを決める**。何を手で書き、何をAIに任せるか、自分で選択する。 |
| 15 | 仕上げ＆振り返り | 最終作品の調整・公開。前期の学びの自己振り返り（記録形式、発表なし）。後期への接続。 |

※授業計画は、学生の理解度や進度に応じて柔軟に調整します。
※QRコード生成は授業内で無料Webサービスを使用します（後半でJavaScriptライブラリを使った自作にも挑戦可能）。

## 12. 予習・復習
- **予習:** 必須ではありません。気になることがあれば事前に調べておくと授業の理解が深まります。
- **復習:** 授業で書いたコードを家でも開いて、自分の言葉でコメントを追加してみましょう。「なぜ動くか」を説明できるようにすることが最大の復習です。

## 13. 評価方法・基準
「動いたか」だけでなく、**コードの意味を自分の言葉で説明できるか**を最重視します。AIの提案を批判的に評価できる力も評価対象です。

| 配分 | 項目 | 内容 |
|---|---|---|
| 30% | コードの理解と説明力 | 自分の作品の各部分について、何をしているか自分の言葉でコメントとして書けるか。 |
| 25% | 完成度・公開実績 | 3つのミニ作品＋最終作品がGitHub Pagesに公開されているか、要件を満たしているか。 |
| 20% | デバッグ・修正力 | バグ修正ワークでの取り組み、エラーへの粘り強い対処、修正記録の質。 |
| 15% | AIの批判的活用 | AIの提案を採用/不採用した理由が記録されているか、AI生成コードの誤りを発見できたか。 |
| 10% | 主体性 | 試行錯誤の継続、エラーへの粘り強さ、授業への積極的な参加。 |

## 14. その他（履修上の注意点）
- **GitHubアカウントとGitHub Studentプラン**が必須です。未取得の学生は授業内でサポートします。
- 開発環境はGitHub Codespacesを使用するため、ローカルPCへのインストールは不要です。ブラウザがあれば学校・自宅どちらでも作業できます。
- プログラミング経験の有無は問いません。自分のペースで一歩ずつ進めれば大丈夫です。
- AIツール（GitHub Copilot等）を意図的に使わない時間を設けます。基礎力を養うために必要なステップです。
- 体調やメンタル面で不安がある場合は、無理せず担当教員または学校の相談窓口に相談してください。
