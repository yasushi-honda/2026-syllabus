「プログラミングは『動くものを作る楽しさ』から始まります。この授業では、AIを活用しながら、一人ひとりのペースで『作って、動かして、理解する』を繰り返します。プログラミング経験の差は気にしなくて大丈夫。AIがみなさんの学習パートナーとなり、つまずいたところを一緒に乗り越えてくれます。大切なのは、『なぜ動くのか？』を自分の言葉で説明できるようになること。コードの裏側にある考え方を理解すれば、どんな言語やツールにも応用が利きます。一緒に『ものづくり』を楽しみましょう！」

授業概要

  • AI（Gemini CLI）を活用したプログラミング入門科目です。VS Codeを開発環境として使用し、Webアプリケーション制作を通じてプログラミングの基礎を学びます。
  • 2〜4年次の混在クラスであることを活かし、AIによる個別最適化学習を取り入れます。経験者はより高度な課題に挑戦し、初学者はAIの支援を受けながら段階的にステップアップします。
  • 毎回「作って動かす」実践を中心に据え、HTML/CSS/JavaScriptの基礎から、Webアプリケーションの構築、GitHubでのバージョン管理、Vercelでの公開まで、現代的な開発フローを体験します。
  • 「AIが書いたコードも、中身を理解することが大切」を基本方針とし、AIの出力を読み解き、修正し、自分のものにする力を養います。

到達目標

知識・理解

  1. HTML/CSS/JavaScriptの基本構文と役割を説明できる。
  2. プログラミングにおける変数、条件分岐、繰り返し、関数の概念を理解している。
  3. Gitによるバージョン管理の基本概念（add/commit/push）を説明できる。
  4. AIを活用したコード生成の利点と注意点（検証の必要性）を理解している。

思考・判断

  1. AIが生成したコードを読み、動作を予測・検証できる。
  2. エラーメッセージを手がかりに、問題の原因を推測し対処を試みることができる。
  3. 要件に対して、どの部分をAIに任せ、どの部分を自分で考えるか判断できる。

スキル・表現

  1. VS Code + Gemini CLIを使って、意図したWebページ・アプリケーションを作成できる。
  2. Git/GitHubを使ってコードを管理し、Vercelで公開できる。
  3. 自分の作品について、使用した技術や工夫した点を説明できる。
  4. ペアやグループで協力して課題に取り組むことができる。

関心・意欲・態度

  1. プログラミングによる「ものづくり」に興味を持ち、主体的に取り組む。
  2. エラーや失敗を学びの機会と捉え、粘り強く試行錯誤する。
  3. 他の学生の作品やアプローチから学ぼうとする姿勢を持つ。

授業計画（全15回）

週 テーマ 主な内容・活動
1オリエンテーション＆初めてのコード授業の目的・進め方・評価方法。VS Code + Gemini CLI環境構築。Live Preview導入。「自己紹介ページ」をHTMLで作成（AIに指示を出して生成→中身を確認する体験）。
2HTML/CSSの基礎HTMLの構造（タグ、要素、属性）。CSSによる装飾の基本（色、フォント、レイアウト）。自己紹介ページの改良。AIに「もっとおしゃれにして」と指示→変更点を読み解く。
3レイアウトとレスポンシブデザインFlexbox/Gridの基本。スマホ対応の考え方。複数ページのサイト構築に挑戦。AIが生成するCSSを読んで、レイアウトの仕組みを理解する。
4JavaScript入門：動きをつける変数、データ型、コンソール出力。ボタンクリックで要素を変化させる。簡単なインタラクション（クリックカウンター等）をAIと一緒に作成。
5条件分岐と繰り返しif文、for文の基本。じゃんけんゲームやクイズアプリの制作。AIに「機能追加して」と指示→追加コードを読み解き、ロジックを理解する。
6関数とモジュール化関数の定義と呼び出し。コードの整理と再利用。制作物のリファクタリング体験（AIに「関数に分けて」と指示→構造の改善を理解）。
7Git入門：セーブ＆ロードバージョン管理の必要性。git init/add/commit。わざとコードを壊す→resetで復元する体験。「安全網があるから挑戦できる」を実感。
8GitHub＆Vercel：世界に公開GitHubアカウント作成、リポジトリ作成、push。Vercel連携で自動デプロイ。自分の作品がURLで世界中からアクセスできる感動体験。
9ミニプロジェクト①：企画と設計個人またはペアで自由制作テーマを決定。AIと対話しながら要件整理・設計。README.mdに企画書を作成。
10ミニプロジェクト②：実装Gemini CLIを活用した高速開発。AIの提案を取捨選択しながら実装。GitHub pushで進捗管理。
11ミニプロジェクト③：改善とデバッグ相互テスト（他の学生に使ってもらう）。フィードバックを基にUI/UX改善。エラー対応・デバッグの実践。
12中間発表：作品共有会ミニプロジェクトの成果発表。工夫した点・苦労した点・AI活用法の共有。相互フィードバック。
13応用：APIとデータ活用（入門）外部APIの概念紹介。天気API等を使ったデータ取得と表示。AIにAPI連携コードを生成させ、データの流れを理解する。
14総合制作：企画〜実装前期の総まとめとして自由制作。これまで学んだ技術の総合活用。AIとの効果的な協働の実践。
15成果発表＆振り返り総合制作の成果発表。プレゼンテーション（デモ＋説明）。相互評価。前期の振り返り。後期への接続。

※授業計画は、学生の理解度や興味、技術の進展に応じて柔軟に変更する可能性があります。

使用テキスト・参考文献

特定のテキストは使用しません。授業内で適宜、資料や参考Webサイトを紹介します。各自のGitHubリポジトリが学習の記録となります。

予習・復習

  • 予習: 必須ではありませんが、次回のテーマについてGemini等に聞いてみると理解が深まります。
  • 復習: 授業で作成したコードをGitHubにpushし、家でも動作確認・改良してみましょう。

評価方法・基準

AIを活用しながらも、コードの意味を理解し、自分の言葉で説明できることを重視します。

30%コードの理解と説明力 — AIが生成したコードを読み解き、動作原理を説明できるか。
25%制作物の完成度 — 要件を満たしているか、動作するか、見た目や使いやすさに配慮があるか。
20%主体的な取り組み — 積極的にAIを試行錯誤しているか、エラーに粘り強く対処しているか。
15%AI活用の工夫 — 効果的なプロンプトの工夫、AIの提案の取捨選択、検証の習慣。
10%協力・共有 — ペアワークへの貢献、他の学生との知識共有、発表での表現力。
