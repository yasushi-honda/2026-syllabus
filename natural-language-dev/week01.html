オリエンテーション＆前年度の進級制作を振り返る

Week 1 木曜 1・2限
テーマ
ガイダンス＋進級制作の再発見
キーワード
GitHub Codespaces / Gemini CLI / 責任あるコード / 振り返り
必要な準備
GitHubアカウント（ログイン可能な状態）・進級制作リポジトリへのアクセス

この回のゴール

  • 授業の全体像（目的・進め方・評価方法）を理解する
  • 前年度（2年次）の進級制作を、今期の通年素材として使うことを理解する
  • Codespaces上で進級制作のコードを開き、久しぶりに自分のコードを読み返す
  • Gemini CLIの起動・OAuth認証を完了する
  • 「当時の自分」と「今の自分」の差を言語化する
  • 「責任あるコード」という概念に出会う

授業の流れ

2コマ連続（90分 + 90分 = 計180分）。途中休憩あり。

1限 前半 約40分

オリエンテーション

教員自己紹介・授業の位置づけ

  • 本田の自己紹介（CTO経験、AI活用の実務、なぜこの授業を作ったか）
  • 「自然言語開発」という科目名の意味——コードを書く言語から、人間の言葉で開発する時代へ

シラバスの説明

  • 全15回の流れを俯瞰（Vibe Coding → SDD → ハーネスエンジニアリング）
  • 評価方法の確認（仕様・設計25%、実装・品質25%、AI活用20%、プロセス15%、責任15%）
  • 「この授業で学ぶスキルは、今のIT業界で最も需要が高まっている分野です」

通年素材の説明

  • 前年度（2年次）の進級制作を、今期の授業を通じて「育て直す」素材として使う
  • 同じコードベースを題材にすることで、成長が可視化される
  • 「壊してもいい、作り直してもいい。大事なのは "なぜそうしたか" を説明できること」
1限 後半 約50分

環境セットアップ＆コードとの再会

GitHub Codespacesで進級制作を開く

  • 進級制作のリポジトリをCodespacesで起動
  • 久しぶりに自分のコードを開き、読み返す
  • 「何がどこにあるか」「なぜこう書いたか」を思い出しながらファイルを辿る

Gemini CLIのセットアップ

  • ターミナルで npx @google/gemini-cli を実行
  • OAuth認証の完了（ブラウザでGoogleアカウントを認証）
  • Gemini CLIの基本操作を確認（質問する、ファイルを読ませる）
Gemini CLIを選ぶ理由: Googleアカウントさえあれば無料で使えるため、全員が同じスタートラインに立てます。後の回でClaude Code等の有料ツールも扱いますが、まずはここから。
休憩 10分
2限 前半 約50分

進級制作の振り返り——過去の自分と対話する

個人ワーク: コードリーディング＆メモ

以下の問いに対して、自分の言葉でメモを書く（Markdown、手書き、どちらでも可）。

Q1 当時の自分は何を考えて、こう書いたのか？
Q2 今読み返して「よく書けている」と思う部分はどこか？
Q3 今読み返して「ここは改善したい」と思う部分はどこか？
Q4 前期末（9月）までに、これをどう育てたいか？

Gemini CLIにコードを読ませて対話してもOK。ただしAIの回答を鵜呑みにせず、自分の判断でメモを書くこと。

2限 後半 約30分

「責任あるコード」の導入

なぜ「責任」がテーマなのか

  • AIがコードを大量に生成できる時代に、開発者に求められるのは「何を作るか」「なぜ作るか」「どう検証するか」を判断する力
  • 「動く」と「正しい」は違う。「速い」と「良い」も違う
  • この授業の15回を通して、「責任あるコード」とは何かを一緒に考えていく

今日の振り返りを「責任」の視点で見直す

  • 進級制作のコード——当時の自分は「責任ある」書き方をしていたか？
  • 答えはYesでもNoでもいい。大事なのは「考えたこと」自体
まとめ 約10分
